# Методы оценки компетенций

Для эффективного использования матрицы компетенций важно иметь прозрачную и объективную систему
оценки. Ниже описаны основные подходы, которые можно применять как для самооценки, так и для внешней
оценки (наставником, руководителем).

## 1. Самооценка

Самооценка позволяет быстро получить первичные данные и отслеживать собственный прогресс.
Рекомендуется использовать одну из трёх шкал или их комбинацию.

### 1.1. Уровневая шкала (модель Dreyfus)

| Уровень | Название   | Описание                                                                                       |
|:-------:|------------|------------------------------------------------------------------------------------------------|
|    1    | Новичок    | Требуются чёткие инструкции, не может действовать самостоятельно                               |
|    2    | Начинающий | Выполняет простые задачи по примерам, нуждается в поддержке                                    |
|    3    | Практик    | Работает самостоятельно над типовыми задачами, находит решения в документации                  |
|    4    | Уверенный  | Глубоко понимает тему, может объяснить другим, выбирает оптимальные подходы                    |
|    5    | Эксперт    | Способен улучшать/оптимизировать существующие решения, обучать других, видит неочевидные связи |

**Применение:** универсальна, подходит для любых навыков (как хард, так и софт).

### 1.2. Шкала частоты применения

Оценивает, как часто вы используете навык на практике.

| Балл | Описание                                  |
|:----:|-------------------------------------------|
|  1   | Никогда не применял                       |
|  2   | Применял редко (раз в полгода или реже)   |
|  3   | Иногда (раз в месяц)                      |
|  4   | Часто (раз в неделю)                      |
|  5   | Постоянно (ежедневно или почти ежедневно) |

**Применение:** хорошо показывает реальную вовлечённость, особенно для технологических навыков.

### 1.3. Шкала уверенности

Оценивает, насколько уверенно вы чувствуете себя при применении навыка.

| Балл | Описание                                        |
|:----:|-------------------------------------------------|
|  1   | Полная неопределённость, не знаю, с чего начать |
|  2   | Нужна постоянная поддержка или подсказки        |
|  3   | Делаю, но часто сомневаюсь в правильности       |
|  4   | Уверен в типовых ситуациях                      |
|  5   | Уверен в любых, даже нестандартных ситуациях    |

**Применение:** полезна для софт-скиллов и сложных тем, где важна не только частота, но и качество.

## 2. Внешняя оценка

Внешняя оценка (наставником, коллегой, руководителем) даёт более объективную картину и позволяет
выявить слепые зоны.

| Метод                            | Описание                                                             | Пример для матрицы                                                                                                                             |
|----------------------------------|----------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------|
| **Тестирование**                 | Проверка знаний через ответы на вопросы или решение небольших задач. | • Для SQL: написать запрос с JOIN и GROUP BY.<br>• Для C#: реализовать паттерн Repository.<br>• Для Git: разрешить конфликт слияния.           |
| **Практические задания (кейсы)** | Выполнение мини-проекта или задачи, приближенной к реальной.         | • Avalonia: создать форму с привязкой данных.<br>• ADO.NET: написать код для выборки из БД.<br>• Командная работа: провести код-ревью примера. |
| **Наблюдение**                   | Оценка в процессе реальной работы или парного программирования.      | Попросить наставника понаблюдать за:<br>• работой с отладчиком;<br>• код-ревью;<br>• оценкой задач на планировании.                            |
| **Проверка артефактов**          | Анализ результатов работы: код, документация, коммиты, пул-реквесты. | Оценить:<br>• качество кода в проектах;<br>• понятность коммитов;<br>• полноту документации к API.                                             |

## 3. Комбинированный подход

Для максимально объективной итоговой оценки рекомендуется комбинировать самооценку и внешние методы.
Пример формулы:

`Итоговая оценка = (Самооценка × 0,3) + (Тест × 0,3) + (Практика × 0,4)`

Где:

- **Самооценка** — по уровневой шкале Dreyfus (1–5).
- **Тест** — результат проверки знаний (например, процент правильных ответов, делённый на 20, чтобы
  привести к 5-балльной шкале).
- **Практика** — оценка выполненного задания наставником (по 5-балльной шкале).

### Пример расчёта для навыка «LINQ (запросы и методы расширений)»

- Самооценка: ставлю себе **3** («делаю самостоятельно типовые запросы»).
- Тест: решено 4 из 5 задач → оценка **4** (4/5 = 0,8; в 5-балльной шкале 4).
- Практика: наставник смотрит код в проекте и ставит **3** («код рабочий, но можно оптимизировать»).

**Итог:**  
(3 × 0,3) + (4 × 0,3) + (3 × 0,4) = 0,9 + 1,2 + 1,2 = **3,3** → округляем до **3**.

## 4. Инструменты для проведения оценки

| Инструмент                                  | Назначение                                                                     |
|---------------------------------------------|--------------------------------------------------------------------------------|
| **Google Forms + Autocrat**                 | Создание опросников для самооценки, автоматический сбор результатов в таблицу. |
| **GitHub Classroom / автоматические тесты** | Проверка практических заданий по программированию.                             |
| **Code Review в Pull Request**              | Получение обратной связи от наставника прямо в коде.                           |
| **Trello / Notion**                         | Трекинг прогресса по навыкам с карточками и чек-листами.                       |

## 5. Рекомендации по применению

1. **Начинайте с самооценки** по уровневой шкале — это даст быстрый срез.
2. **Для ключевых навыков** (например, LINQ, SQL, Git) проводите **ежемесячные мини-тесты** или
   практические задания.
3. **Фиксируйте результаты** в Excel-таблице (см. `competence_matrix.xlsx` или
   `competence_matrix.csv`) с датами и примечаниями — это позволит отслеживать динамику.
4. **Используйте внешнюю оценку** при наличии наставника или в рамках учебных проектов.
5. **Не стремитесь оценить сразу всё** — выберите 5–7 приоритетных навыков на месяц и работайте с
   ними.

Эти методы помогут сделать процесс развития системным и прозрачным, а матрица компетенций станет не
просто списком, а работающим инструментом роста.
